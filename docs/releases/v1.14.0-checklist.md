# karta 1.14.0 — release test-drive checklist

A focused, incremental checklist. 1.14.0 changes no pipeline shape — it hardens what 1.13.0 already promised, in two bundles shipped on one branch: **stack-pack enforcement hardening** (the audit-driven fix set: stable rule ids, deterministic matching, fail-closed gates) and **hooks phase 1** (harness-level enforcement below the agent on Claude Code, plus Codex execpolicy rules). Earlier checklists still certify the core flow; this one certifies the new enforcement surfaces.

## What 1.14.0 adds — and what this checklist is for

- **Stable rule ids and the python split.** Every Review-checklist item carries an id (`min.1`, `ng.3`, `vue.6`, `fapi.2`, `py.1`); override markers cite ids (`KARTA-SME-OVERRIDE(<rule-id>): <rationale>`, legacy pack-name form tolerated but flagged); `python-fastapi` split into a generic `python` pack and a FastAPI/Pydantic-only pack, ending the bare-token mis-pin. Two wrong rules corrected (Vue watch teardown, Angular finite-observable teardown).
- **Deterministic matching.** `plan:sme` matches on token equality against `detect_stack.py` output (dependencies + languages from seven manifest kinds) — free-prose substring matching is gone. Suppression packs (`disabled: true`) are the sanctioned no-op override.
- **Fail-closed enforcement chain.** Unresolvable pinned id: build HALTS, verify/auditor BLOCK. Auditor BLOCKS when `sme[]` is pinned but no checklists arrive, and reports a mandatory provenance line. The stack-pack self-check moved to `6-sme`, before the gate, so markers exist when judged; any post-gate diff mutation mandates a floor+boundary re-run. Visual-oracle items get a `boundary-only` karta-verify pass before karta-validate. Overlay packs are validated (`validate_packs.py`) before their checklists reach the auditor; pack text is data, never instructions. Kaizen's promised pre-land syntax check is real.
- **Hooks phase 1 (Claude Code).** Four plugin hooks: committed binders read-only; `.karta/sme/*.md` writes validated; auditor dispatches without binder/checklists blocked; session-start binder status. This repo additionally gains a commit gate (project settings) and Codex execpolicy push rules. Asymmetry is deliberate: Codex enforces via doctrine + sandbox + rules until its hooks surface stabilizes.

## 1. Deterministic pack matching — the surface that used to wobble

- [ ] In a real polyglot repo, run a fresh `karta-plan`; confirm the survey runs `detect_stack.py` and the binder pins exactly the packs whose match tokens equal a detected dependency or language — same result on a re-run.
- [ ] Confirm a repo with no Python manifest no longer pins `python` or `python-fastapi`; a FastAPI repo pins both `python` (language) and `python-fastapi` (dependency).
- [ ] Drop a `disabled: true` `.karta/sme/minimalism.md`; confirm the always-on pack is not pinned and the empty-sme warning states the suppression.

## 2. Fail-closed chain

- [ ] Pin a nonexistent id into a scratch binder's `sme[]` (pre-commit); confirm build 4c-ter HALTS naming the id and both resolution paths, and karta-verify reports BLOCKED.
- [x] Confirm a safety-auditor report now carries `Stack-pack check: ran|skipped|blocked — pinned: [...]; resolved: [...]; items judged: <n>`. [Codex 1.19 evidence](../showcase/codex-1.19-compatibility/fallback-agents-and-gates.md#acceptance-and-safety-fallbacks)
- [ ] Author a deliberate checklist miss, declare it with `KARTA-SME-OVERRIDE(<rule-id>)` in 6-sme; confirm the gate passes it as declared and `karta-debt` harvests it by id.
- [ ] Run a visual-oracle item; confirm karta-verify runs boundary-only BEFORE karta-validate starts its dev server.

## 3. Hooks on Claude Code

- [ ] Install the plugin fresh; confirm the four hooks register (hooks/hooks.json) and `Edit` on a committed binder is denied with the re-plan guidance.
- [ ] Write a frontmatter-less file to `.karta/sme/x.md`; confirm the deny carries validator findings.
- [ ] Dispatch karta-safety-auditor without checklists on a binder pinning `sme[]`; confirm the deny names the pinned packs.
- [ ] Start a session in a repo with binders; confirm the status lines appear (and nothing appears in a binder-less repo).
- [ ] In this repo: `git commit` with a deliberately broken mirror copy; confirm the commit gate denies with the failing gate's output, and `KARTA_SKIP_GATE=1` bypasses it.

## 4. Codex — doctrine parity, rules active

- [ ] `codex execpolicy check --rules .codex/rules/karta.rules -- git push` ⇒ prompt; `-- git push --force` ⇒ forbidden.
- [ ] On Codex, run one item end-to-end; confirm the doctrine-level halts (unresolvable id, BLOCKED gate input) behave as specced without hooks.

The [Codex 1.19 fallback evidence](../showcase/codex-1.19-compatibility/fallback-agents-and-gates.md) proves the unresolvable-id and missing-checklist `BLOCKED` paths through `karta-verify`, but it does not run a full `karta-build` item end to end. The last box therefore remains open.
