# karta 2.20.0 — release test-drive checklist

Karta 2.20.0 makes the tested Codex support boundary explicit. The installed Codex plugin passed the recorded fallback-agent, gate, Kaizen, and Plannotator scenarios. This release also includes the macOS binder-path guard fix found during that compatibility work.

## What 2.20.0 certifies

- **Fallback agents and gates:** installed-plugin sessions loaded bundled agent instructions, dispatched read-only acceptance and safety reviewers, and failed closed when stack-pack input was missing.
- **Kaizen:** absent and disabled switches were no-ops; direct mode selected the expected packs; delivery mode seeded the binder-pinned packs and committed only on the supplied integration branch.
- **Plannotator:** capability detection, browser annotations, precise change mapping, ambiguous-feedback return, binder validation, and the no-implicit-commit rule worked in the live round-trip.
- **macOS binder paths:** the immutability guard now treats equivalent `/var` and `/private/var` paths as the same binder path.

## Pre-flight (done for 2.20.0)

- [x] Version is `2.20.0` in all four plugin manifests.
- [x] Plugin integrity, shared-copy parity, Codex agent sync, and Codex skill projection checks pass on the tracked tree.
- [x] Binder `codex-119-compatibility` delivered 5/5 items and is archived.
- [x] No compatibility scenario pushed a branch or opened a pull request.

## Evidence

Start with the [Codex compatibility result](../showcase/codex-1.19-compatibility/README.md). It links to the detailed fallback-agent, Kaizen, and Plannotator records and states the enforcement boundary for each feature.

The live runs used Codex CLI 0.143.0 for the fallback and Plannotator scenarios. Kaizen used 0.143.0 for shell checks and 0.144.0-alpha.4 for its installed-plugin sessions.

## Known limits

- Codex does not provide Karta's Claude Code hooks. The evidence does not claim hook parity or a boundary narrower than the active Codex sandbox.
- Repo-local Codex projections are byte-synced, but the live feature scenarios ran through the installed plugin and were not repeated in repo-local mode.
- Fallback-gate evidence and Plannotator evidence carry explicit human acceptance waivers. The fallback waiver preserves honest retry history; the Plannotator waiver records an automated contract-test gap after the live flow passed.
