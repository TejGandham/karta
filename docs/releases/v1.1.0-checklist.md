# karta 1.1.0 — release test-drive checklist

A focused, incremental checklist. 1.1.0 keeps the 1.0.0 pipeline unchanged and adds two things: **Codex CLI compatibility** (karta now installs and runs as a Codex plugin, not just a Claude Code plugin) and the **opt-in automatic doc-gardner**. This checklist certifies those two — plus a quick regression pass to confirm the 1.0.0 pipeline still behaves — without repeating the full [1.0.0 test-drive](v1.0.0-checklist.md), which still applies to the core flow.

## What 1.1.0 adds — and what this checklist is for

- **Codex CLI compatibility.** Every skill, agent, and manifest is now a valid artifact on both runtimes. Codex artifacts (`.agents/skills/` mirror, `.codex/agents/*.toml`, bundled `*.agent.md`) are *generated* from the Claude-native canonical sources and drift-guarded, so the two runtimes never diverge. The gate agents run automatically on a Codex plugin install even though Codex plugins cannot register subagents.
- **Opt-in automatic doc-gardner.** A single switch (`.karta/doc-gardner.json`) makes every delivery end by auto-correcting drifted docs as one labeled, revertible commit. All-or-nothing: opted in, drift is fixed silently; opted out, it never runs.

It does **not** change the gates, verdicts, hatch, or git-native resume — those are still as certified at 1.0.0. The one thing never exercised before this release: a clean install **on Codex CLI**, run against a real repo.

## 1. Clean install on Codex CLI — the gate never run before this release

- [ ] In a throwaway environment, add the repo marketplace and install karta on **Codex CLI** (the `.agents/plugins/marketplace.json` local marketplace, or the published source).
- [ ] Confirm Codex auto-discovers all seven skills from `.agents/skills/<name>/SKILL.md` — including `karta-doc-gardner`.
- [ ] Confirm the gate agents run **automatically** on the plugin install: invoke `karta-verify`'s path and confirm it spawns a **read-only** subagent from the bundled `references/*.agent.md` (the plugin-install fallback, since Codex plugins can't register subagents).
- [ ] If `.codex/agents/*.toml` are present (the registered-subagent path), confirm `karta-acceptance-reviewer` and `karta-safety-auditor` load with `sandbox_mode = "read-only"`, and `karta-doc-gardner` with `workspace-write`.
- [ ] Confirm `AGENTS.md` is picked up as project instructions.

## 2. Regression — the 1.0.0 pipeline is unaffected

The dual-runtime refactor touched skill files, so re-confirm the core flow on **Claude Code**:

- [ ] On a small real repo, run `karta-plan` → `karta-deliver` end to end and confirm it still builds in parallel waves onto `karta/<slug>/integration`, gates each item, and ends at the integration branch — no PR, no push.
- [ ] Confirm at least one gate path still fires as documented (a kickback to build on DEVIATION, or a safety-auditor escalation). Full coverage lives in the [1.0.0 checklist](v1.0.0-checklist.md) §3.

## 3. doc-gardner — off by default, all-or-nothing when on

- [ ] **Off by default:** with no `.karta/doc-gardner.json`, run a delivery and confirm the gardner phase never runs and no `docs: gardner` commit appears.
- [ ] **Turn it on:** add `.karta/doc-gardner.json` → `{ "enabled": true }`. Introduce doc drift (rename a symbol a doc references, or leave a "will add" note for something now landed), run `karta-deliver`, and confirm a single `docs: gardner <slug>` commit lands on the integration branch with the drift corrected — no advisory report, no waive, no halt.
- [ ] **Scope is recomputed live:** confirm both a doc added in an earlier delivery and newly-changed code are swept this run (nothing about *what* to garden is cached).
- [ ] **Reviewable and revertible:** `git show` the gardner commit to inspect exactly what changed; confirm `git revert <sha>` cleanly undoes a correction you disagree with.
- [ ] **Stays in its lane:** confirm it edited only prose docs — never code, tests, the binder, or refs — and left accurate prose, design rationale, and dated archival docs (`docs/specs/YYYY-MM-DD-*`) alone.
- [ ] **Ad-hoc run:** invoke `karta-doc-gardner` directly ("garden the docs") with the switch **off**, and confirm it corrects the working tree and hands the edits back for you to commit — the switch only governs the automatic delivery path.

## 4. Cleanup

- [ ] After a Codex run, confirm worktrees, `karta/<slug>/*` branches/tags, and `refs/karta/<slug>/*` refs clean up or are clearly listed for `clear` — same as the Claude path.
- [ ] Confirm no `.karta/` state (including a stray `doc-gardner.json` you added for testing) is left behind unintentionally.

## If something fails

The 1.0.0 failure posture is unchanged: karta surfaces every halt with a call to action and never opens a PR or pushes on its own. Two 1.1.0-specific notes:

- A doc-gardner mis-correction is just a commit — `git revert` it, or drop it before merging the integration branch. There is no human gate to re-add; the reviewable commit *is* the gate.
- If a Codex artifact looks wrong, it is generated — fix the canonical source (`skills/`, `agents/`) and re-run the generators (`scripts/sync_codex_skills.py`, `scripts/sync_codex_agents.py`); never hand-edit the `.agents/` or `.codex/` projections. See [AGENTS.md](../../AGENTS.md).
