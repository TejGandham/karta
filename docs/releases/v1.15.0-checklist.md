# karta 1.15.0 — release test-drive checklist

A focused, incremental checklist. 1.15.0 is the evidence-and-dogfood release on top of 1.14.0's hardening: the stack-pack system now has its first recorded proof, a deterministic kaizen seeding contract validated by a real consumer delivery, and karta itself dogfoods kaizen under a scoped policy. The [1.14.0 checklist](v1.14.0-checklist.md) still certifies the enforcement chain.

## What 1.15.0 adds — and what this checklist is for

- **First recorded benchmark result.** `benchmarks/sme/results/2026-07-06-date-picker.md`: n=3 per arm, all six runs passed acceptance with zero new dependencies either way; packs-on halved the median diff (8 added LOC vs 14). `run.py`'s new-deps metric now counts only manifest dependency pins (code lines with `==`/`>=` no longer register) and ships a self-test.
- **Kaizen seeding is mode-deterministic.** "Every pack the project uses" is now defined: delivery mode seeds exactly the binder's pinned `sme[]` (seed what the delivery built and gated against — never re-derive); direct mode derives the way `plan:sme` does, via `detect_stack.py` token equality. Validated live: parchmark's first seed (three packs, 1.14.0 numbered ids, all pass `validate_packs.py`) is exactly this behavior.
- **The flagship example engages the expert layer.** `example-binder.json` is now a Vue 3 app pinning `["minimalism", "vue"]`, with item contracts written in the vue pack's idioms — the imitation artifact demonstrates domain guidance instead of showing packs with nothing to say.
- **Kaizen dogfood on karta itself (scoped).** `.karta/kaizen.json` enabled; `.karta/sme/minimalism.md` is a managed byte-identical shadow guarded by `validate_plugin.py` (drift = gate failure, discard-or-promote-upstream); `karta-house-skill-authoring` is the repo's own always-on non-coding pack (`house.1-3`: mirrors travel with canonical edits, scripts ship `--self-test`, no dangling pointers after moves); the commit gate validates `.karta/sme/` overlay packs; policy in AGENTS.md.
