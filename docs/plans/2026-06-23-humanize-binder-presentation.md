# Humanize binder & work-item presentation — implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give every binder and work item a human-friendly `title` and a plain-language `summary`, authored by karta-plan and surfaced by Karta Watch, so a person can read what a run is doing without decoding kebab-case slugs.

**Architecture:** Authored-into-the-artifact, not generated at view time. The binder JSON gains the fields (required, enforced by `validate_binder`); karta-plan writes them through the karta-plainlanguage standard; the read-only Karta Watch page leads with them and falls back to a Title-Cased slug for binders made before this feature. No engine, git-derivation, or layout changes.

**Tech Stack:** PEP 723 zero-dependency stdlib Python (`validate_binder.py`, `serve_status.py`), a JSON Schema (`binder-schema.json`), vendored Vue 3 in `serve_status.py`, Markdown skill docs.

## Global Constraints

- The binder is **always a karta-generated artifact**, never hand-written — so the new fields are **required** in the schema and `validate_binder` rejects a binder missing them.
- `serve_status.py` stays **self-contained and zero-dependency**: no external URLs, no new deps, no view-time prose generation. The page is **read-only** and never halts on missing fields — it degrades to a Title-Cased slug.
- `binder-schema.json` keeps `additionalProperties: false` at every level; new fields are added explicitly.
- Edit only the **canonical** sources under `skills/`; the `.agents/` and `plugins/karta/` mirrors are regenerated by `scripts/sync_codex_skills.py` (do not hand-edit mirrors).
- Field names, verbatim: binder gains `title` + `summary`; work item gains `summary` (its existing `title` is promoted to the headline). Both `summary` fields and the binder `title` are `string`, `minLength: 1`, required.
- Self-test counts must stay green: `validate_binder.py --self-test` and `serve_status.py --self-test`, plus the four pre-commit gates.

- [ ] **Step 1: Carry the binder-level human fields through `_enrich`**

In `_enrich`, after building `wi_by_slug`, also index the raw binder docs by slug and copy the human fields onto each state binder; and set per-item `title`/`summary`. Replace the item loop body so it sets `title`/`summary` (keep `desc` for back-compat-free removal):

```python
    by_slug = {b["slug"]: b for b in binders}
    for ob in state["binders"]:
        src = by_slug.get(ob["slug"], {})
        ob["title"] = src.get("title")
        ob["summary"] = src.get("summary")
        ob["motivation"] = src.get("motivation")
        items = wi_by_slug.get(ob["slug"], {})
        for d in ob["items"]["detail"]:
            wi = items.get(d["id"], {})
            oracle = wi.get("oracle", {}) or {}
            otype = "opt-out" if oracle.get("opt_out") else oracle.get("type", "unit")
            assertions = oracle.get("assertions") or []
            d["title"] = wi.get("title")
            d["summary"] = wi.get("summary")
            d["oracle"] = otype
            d["assert"] = assertions[0] if assertions else None
            d["cmd"] = oracle.get("command")
            d["deps"] = wi.get("depends_on", []) or []
    return state
```

- [ ] **Step 2: Add a Title-Case helper and use the human fields in `mkBinder`**

function titleCase(slug) {
  return String(slug || '').split('-').filter(Boolean)
    .map(w => w[0].toUpperCase() + w.slice(1)).join(' ');
}
