# Card errata — 2026-07-17

This ledger records every place where the `bench-probe-buildout` build deliberately
departs from what a vector card says. Each row names the card, quotes what the card
text asks for, states what actually shipped, and gives the reason. The point is to
reconcile card and code on purpose: a probe that implements a subset flips only its
own card's `probe_status` frontmatter — no card procedure, metric, or spec semantics
is rewritten to match the code. Read a row as "the card still says X; the code does Y,
and here is why that is correct." Fix the card or the code from here, never let them
drift silently.
